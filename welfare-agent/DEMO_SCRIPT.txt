[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture diagram to see how it works under the hood.
First, when a user sends a query or demographic details, the input passes through the Security Checkpoint node, which scrubs sensitive PII and filters out malicious injections.
(pause)
Once verified, the input reaches our root_agent Coordinator. This orchestrator dynamically delegates queries to our specialized sub-agents.
For instance, the user_profile_agent formats citizen profiles.
The scheme_discovery_agent searches our localized schema databases.
The eligibility_evaluator_agent matches profiles against specific rules and calculates score cards.
The scheme_comparison_agent compares financial benefits side-by-side.
The document_verifier_agent checks uploaded certificates for completeness.
The fraud_detection_agent scans suspicious links and alert messages.
The life_event_prediction_agent predicts upcoming benefit milestones.
And finally, the welfare_score_agent tracks the citizen's claimed versus potential financial aid.
These agents pull from a SQLite database proxy and interface with external resources via our MCP Server.
Also, when connecting with NGOs for direct assistance, the workflow utilizes a Human-in-the-Loop review process where NGO representatives approve requests from their dashboard.
